# QR-идентификация студента для физической культуры

Прототип встроен в исходный код личного кабинета Московского Политеха и добавляет QR-сценарий отметки посещения на странице физической культуры.

## Что реализовано

- Студент на странице физкультуры может открыть QR-код для отметки посещения.
- QR-код живет 20 секунд, после чего автоматически обновляется.
- Под таймером показана убывающая линия времени жизни QR.
- QR-блок можно скрыть красной кнопкой-крестиком.
- Преподаватель получает вкладку `QR-отметка` в интерфейсе физической культуры.
- Преподаватель может открыть камеру, отсканировать QR-код студента и выставить отметку.
- Количество баллов преподаватель не выбирает: прототипный backend сам назначает балл.
- Для локальной проверки в demo/dev-режиме преподавательский интерфейс открыт всем авторизованным пользователям.

## Как запустить

Из корня проекта:

```powershell
docker compose up -d --force-recreate --remove-orphans lk-local
```

Открыть в браузере:

```text
http://localhost:3000
```

Посмотреть логи:

```powershell
docker compose logs -f lk-local
```

Остановить:

```powershell
docker compose stop lk-local
```

## Где смотреть

Студент:

```text
http://localhost:3000/#/physical-education/student
```

Преподавательский интерфейс для локального теста:

```text
http://localhost:3000/#/physical-education/main
```

Если браузер держит старую версию, один раз сделайте `Ctrl + F5`. В dev-режиме приложение также снимает старый service worker, чтобы localhost не подсовывал закэшированный HTML.

## Основные измененные места

- `src/pages/pe-student` - подключение QR-блока на странице студента.
- `src/features/physical-education/student/pe-student-qr` - UI студенческого QR-кода.
- `src/pages/teacher-physical-education` - вкладка преподавателя `QR-отметка`.
- `src/pages/teacher-physical-education/qr-scanner` - сканер QR-кода преподавателя.
- `src/shared/api/physical-education` - типы и методы QR API.
- `src/entities/pe-teacher/model.ts` - dev-only выдача тестового доступа преподавателя.
- `src/shared/lib/dev-pe-access.ts` - demo-переключатель доступа к преподавательскому интерфейсу.
- `vite.config.ts` - локальный mock `/qr-api`, proxy `/pe-api` и корректная отдача `index.html`.
- `docker-compose.yml` - сервис `lk-local` для запуска через Docker.

## Как работает прототипный QR API

В dev-режиме QR API живет внутри Vite middleware по адресу `/qr-api`.

Основные методы:

- `POST /qr-api/qr/create` - создать временный QR-токен.
- `POST /qr-api/qr/scan` - проверить QR-токен.
- `POST /qr-api/points/mark` - отметить посещение и назначить балл.

Состояние прототипа хранится в памяти dev-сервера. После перезапуска контейнера созданные QR-токены и отметки очищаются.

## Нагрузочные ограничения

Текущий mock подходит для демонстрации UX и проверки сценария, но не является production-архитектурой.

Для 10 000 студентов генерация QR сама по себе не проблема: QR-токен маленький, живет 20 секунд, а операция создания дешевая. Но в production нельзя хранить токены только в памяти одного Vite/Node-процесса.

Для реальной нагрузки нужно вынести QR API в backend-сервис и использовать:

- Redis или другую быструю TTL-storage для QR-токенов;
- атомарное погашение токена, чтобы один QR нельзя было использовать дважды;
- rate limiting на создание QR;
- аудит отметок в базе данных;
- горизонтальное масштабирование backend-инстансов;
- мониторинг ошибок и задержек.

В такой схеме 10 000 студентов не должны быть критичной нагрузкой: активных QR одновременно будет меньше общего числа студентов, а TTL-хранилище хорошо подходит для короткоживущих токенов.

## Как убрать demo-доступ преподавателя

Перед production-интеграцией нужно удалить временную выдачу преподавательского доступа. Это важно: сейчас она специально открывает преподавательскую физру всем авторизованным пользователям на localhost/demo.

Что убрать:

1. Удалить файл `src/shared/lib/dev-pe-access.ts`.
2. В `src/entities/pe-teacher/model.ts` убрать ветку с `isDemoPeTeacherAccessEnabled()` и оставить только реальный запрос `peApi.getTeacher(...)`.
3. В `src/entities/menu/model/index.ts` убрать `getDevPeTeacherRoute`, `getStudentRoutesWithDevPeTeacher`, `getStudentAllRoutesWithDevPeTeacher` и вернуть обычные `studentRoutes()` / `studentHiddenRoutes(...)`.
4. В `src/app/routing/routes/pages/all-pages.ts` убрать подмешивание `physical-education-teacher` в студентские страницы.
5. В `src/shared/routing/routes/index.ts` убрать подмешивание `physical-education-teacher` в студентские маршруты.

После этого доступ к преподавательскому интерфейсу снова должен определяться только реальными правами из backend/API.
